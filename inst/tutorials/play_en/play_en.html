Object-Oriented Programming

Object-Oriented Programming (OOP) organises and encapsulates data through classes and objects. Many widely known languages — C++, Java, Python — adopt OOP, and R does the same.

  • Class: a user-definable structure of an object, together with its properties and methods.
  • Object: a concrete instance of a class — “China” is to “country”, “R” is to “letter”.
  • Method: a command defined within a class.
  • Property: an attribute of an object — like hair colour for a person.

OOP languages also typically share encapsulation, polymorphism, inheritance, and safety.

Polymorphism lets one command act differently on different object types. The most familiar example is summary():

summary(wvs7)
summary(wvs7$age)

Functions

A function can be thought of as an imperative sentence with the adverbial postposed:

<function_name>(<target_data>, <condition_1>, <condition_2>, ...)

The OOP creed: do not touch the data; let the function act on it.

light <- function(finger){
  shadow <- finger + 5
  shadow
}
handShadow <- light(finger = 3)
handShadow

The Assignment Arrow <-

<- is R’s standard assignment operator, equivalent to assign():

<name> <- <object>

aValidObject <- 1:5
aValidObject
## [1] 1 2 3 4 5

R supports four assignment symbols: assign(), <-, <<-, =. <- is preferred — direction is intuitive, and it does not collide with = or ==. Keyboard shortcut: Alt + - (Windows/Linux), Option + - (macOS).

a <- 12
25 -> b   # the rightward arrow also works

Naming Rules

Object names must satisfy:

  1. Not begin with a digit (incorrect: 1stday).
  2. Contain no special characters except . and _ (incorrect: M&M).
  3. Are case-sensitive (X != x).
  4. Avoid overwriting built-in names (avoid list <- c(1:5)).
  5. Convey meaning.

Records (Data Storage)

Vectors

c() builds vectors. A vector’s elements must share the same type.

Numeric Vectors

vec_integer <- c(1, -2, NA)
vec_double  <- c(1.5, -2.34, 1/3)

NA stands for not available — R’s representation of a missing value.

Character Vectors

vec_chr <- c("R", "is", "useful.")
vec_chr
## [1] "R"       "is"      "useful."

Factor Vectors

A factor stores a categorical variable with a fixed set of allowed values (its levels).

vec_fac  <- factor(c(1, 2, 2, 3))
vec_ord  <- ordered(c(1, 2, 2, 3))
vec_fac2 <- factor(c(1, 2, 2, 3),
                   levels = c(3, 2, 1),
                   labels = c("Apple", "Pear", "Orange"))

vec_fac
## [1] 1 2 2 3
## Levels: 1 2 3
vec_ord
## [1] 1 2 2 3
## Levels: 1 < 2 < 3
vec_fac2
## [1] Orange Pear   Pear   Apple 
## Levels: Apple Pear Orange

Date and Time Vectors

R offers two representations of time:

  • as.POSIXct() — numeric storage, seconds since 1970-01-01.
  • as.POSIXlt() — list storage, each component as a separate element.
ct <- as.POSIXct("2026-04-13 10:11:12")
lt <- as.POSIXlt("2026-04-13 10:11:12")

unlist(ct)
## [1] "2026-04-13 10:11:12 CST"
unlist(lt)
##    sec    min   hour   mday    mon   year   wday   yday  isdst   zone gmtoff 
##   "12"   "11"   "10"   "13"    "3"  "126"    "1"  "102"    "0"  "CST"     NA

The Swiss-army knife for dates is lubridate:

ymd("20221016")
## [1] "2022-10-16"
mdy("10-16-2022")
## [1] "2022-10-16"
dmy("16/10/2022")
## [1] "2022-10-16"
ymd_hms("2022-10-16 09:00:00", tz = "Etc/GMT+8")
## [1] "2022-10-16 09:00:00 -08"

Matrices, Arrays, Lists, and Data Frames

Matrices were introduced in drhur("calculate"). Arrays generalise matrices to higher dimensions:

vector1 <- c(5, 9, 3)
vector2 <- c(10, 11, 12, 13, 14, 15)

result <- array(c(vector1, vector2), dim = c(3, 3, 2))
result
## , , 1
## 
##      [,1] [,2] [,3]
## [1,]    5   10   13
## [2,]    9   11   14
## [3,]    3   12   15
## 
## , , 2
## 
##      [,1] [,2] [,3]
## [1,]    5   10   13
## [2,]    9   11   14
## [3,]    3   12   15

A list is a heterogeneous container; elements may differ in type or length:

ls_monks <- list(
  name   = c("Wukong Sun", "Sanzang Tang", "Wuneng Zhu", "Wujing Sha"),
  power  = c(100, 20, 90, 40),
  buddha = c(TRUE, TRUE, FALSE, FALSE)
)

ls_monks
## $name
## [1] "Wukong Sun"   "Sanzang Tang" "Wuneng Zhu"   "Wujing Sha"  
## 
## $power
## [1] 100  20  90  40
## 
## $buddha
## [1]  TRUE  TRUE FALSE FALSE

A data frame is a special list (or matrix) where each column is a variable of equal length and each row is an observation:

df_toy <- data.frame(
  female = c(0, 1, 1, 0),
  age    = c(29, 39, 38, 12),
  name   = c("Iron Man", "Black Widow", "Captain Marvel", "Captain America")
)

df_toy
