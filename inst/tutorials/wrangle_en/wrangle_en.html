Data Wrangling with dplyr

The tidyverse is a family of R packages sharing a common philosophy and data structure. Its core components — dplyr, ggplot2, tidyr, tibble — work together seamlessly.

The Pipe

R’s native pipe |> connects sequential operations on the same object. Each step occupies its own line, in the order it is performed.

# Traditional nested style: read inside-out, hard to follow
eatDumpling <-
eat(
  dip(
    cook(
      fill(
        mix(
          meat,
          with = c(salt, soy_sauce, green_onion, ginger)
          ),
        with = wrapper
        ),
      in_water = boiled_water
      ),
    in_sauce = vinegar)
  )
# Pipe style: read top-down, logic is clear
eatDumpling_pipe <-
mix(meat, with = c(salt, soy_sauce, green_onion, ginger)) |>
  fill(with = wrapper) |>
  cook(in_water = boiled_water) |>
  dip(in_sauce = vinegar) |>
  eat()

Keyboard shortcut for |> (or %>%): Ctrl + Shift + M (Windows / Linux), Cmd + Shift + M (macOS).

Selecting Variables

select() keeps the listed columns:

library(tidyverse)

# Pick five variables of interest
select(wvs7, equalIncentive, incomeLevel, confidence_gov, education, country)

Helpers like starts_with(), ends_with(), and matches() operate on column-name patterns. For example, the three corruption-perception variables all start with corruption:

# Pick all variables starting with "corruption"
select(wvs7, starts_with("corruption"))

To drop variables instead of keeping them, prepend -. We can combine starts_with() with - to drop a subset:

# Keep corruption_local and corruption_civil, drop corruption_state
select(wvs7, starts_with("corruption"), -corruption_state)

Renaming Variables

rename() uses the form new_name = old_name:

rename(wvs7, nationality = country)

Filtering by Value

While select() keeps columns, filter() keeps rows that satisfy a logical condition.

# Respondents from Japan with high confidence in government
wvs7 |>
  filter(country == "Japan", confidence_gov >= 3)

Arranging Rows

arrange() sorts by the listed variables (ascending by default; use desc() for descending).

wvs7 |>
  select(age, confidence_gov, education, incomeLevel) |>
  arrange(age)

Modifying and Creating Variables

mutate() modifies existing variables or creates new ones.

# Average of two confidence variables
wvs7 |>
  mutate(avg_confidence = (confidence_gov + confidence_court) / 2) |>
  select(country, confidence_gov, confidence_court, avg_confidence)

Merging Datasets

Direct Merging

  • bind_rows() stacks two data frames vertically. Mismatched columns are filled with NA.
  • bind_cols() stacks two data frames side by side. The two must share row count.
wvs7_jp <- filter(wvs7, country == "Japan")
wvs7_kr <- filter(wvs7, country == "South Korea")

bind_rows(wvs7_jp, wvs7_kr)
wvs7_conf  <- select(wvs7, starts_with("confidence"))
wvs7_trust <- select(wvs7, starts_with("trust"))

bind_cols(wvs7_conf, wvs7_trust)

Index Merging (Joins)

Index merging combines two datasets on a shared key.

Structural consistency

Index merging requires that the index variables follow the same coding scheme. The variables need not contain identical sets of values, but their meanings must align.

We construct two demonstration datasets:

  1. wvs7_eq — individual-level data on equality perceptions, restricted to Japan and South Korea.
  2. country_info — country-level summaries of demographic variables, restricted to Japan and Thailand.
# Individual-level data: Japan and South Korea
wvs7_eq <- select(wvs7, country, equalIncentive, female, education) |>
  filter(country %in% c("Japan", "South Korea"))

# Country-level data: Japan and Thailand
country_info <- filter(wvs7, country %in% c("Japan", "Thailand")) |>
  group_by(country) |>
  summarise(across(female:education, \(x) mean(x, na.rm = TRUE))) |>
  ungroup() |>
  rename(nation = country)

wvs7_eq
country_info

dplyr provides four join types:

  • left_join() — keep all rows of the left dataset; unmatched rows on the right become NA.
  • right_join() — mirror image: keep all rows on the right.
  • inner_join() — keep only rows whose key appears in both.
  • full_join() — keep all rows from either side.
# Run all four join types
wvs7_lj <- left_join(wvs7_eq,  country_info, by = join_by(country == nation))
wvs7_rj <- right_join(wvs7_eq, country_info, by = join_by(country == nation))
wvs7_ij <- inner_join(wvs7_eq, country_info, by = join_by(country == nation))
wvs7_fj <- full_join(wvs7_eq,  country_info, by = join_by(country == nation))

# Compare which countries each retained
wvs7_lj$country |> unique()
wvs7_rj$country |> unique()
wvs7_ij$country |> unique()
wvs7_fj$country |> unique()

Section Exercise

Build a single dplyr pipeline on wvs7 that:

  1. Creates a new variable avg_confidence — the mean of confidence_gov and confidence_court.
  2. Keeps respondents from Japan, aged strictly between 40 and 60, with avg_confidence above 3.
  3. Sorts the result by avg_confidence in descending order.
  4. Returns only age, incomeLevel, equalIncentive, and avg_confidence.
wvs7 |>
  mutate(avg_confidence = (confidence_gov + confidence_court) / 2) |>
  filter(
    country == "Japan",
    age > 40,
    age < 60,
    avg_confidence > 3
  ) |>
  arrange(desc(avg_confidence)) |>
  select(age, incomeLevel, equalIncentive, avg_confidence)

Summary

  • Data exploration: nrow(), ncol(), glimpse(), names(), unique(), table()
  • Data wrangling
    • Pipe: |>
    • Select columns: select()
    • Rename: rename()
    • Filter rows: filter()
    • Arrange: arrange()
    • Modify or add: mutate()
  • Data merging
    • Row-binding: bind_rows()
    • Column-binding: bind_cols()
    • Joins: left_join(), right_join(), inner_join(), full_join()
