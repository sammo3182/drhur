Basic Operations

R’s arithmetic symbols match those of mathematics: +, -, *, /.

# The four basic operations
1 + 2
## [1] 3
3 - 4
## [1] -1
5 * 6
## [1] 30
7 / 8
## [1] 0.875
# Compound expressions
(2 + 4) / (5 - 3)
## [1] 3

A note on whitespace

Spaces between numbers and operators are added for readability — R itself is indifferent to whitespace. The expression below runs perfectly:

(2+4) /  (  5-   3        )
## [1] 3

For inherited or third-party code, packages such as styler and air can reformat code in a single click.

R also supports more complex arithmetic: powers (^), modulo (%%), logarithms (log(), log10()), exponentials (exp()), square root (sqrt()), and absolute value (abs()).

10^2
## [1] 100
5 %% 3      # remainder
## [1] 2
log(11)
## [1] 2.397895
log10(100)
## [1] 2
exp(12)
## [1] 162754.8
# Two ways to take a square root
9^(1/2)
## [1] 3
sqrt(9)
## [1] 3
# Absolute value
abs(-9)
## [1] 9

Trigonometric functions:

sin(pi)
## [1] 1.224647e-16
cos(pi)
## [1] -1
tan(pi / 2)
## [1] 1.633124e+16
acos(1)
## [1] 0
asin(1)
## [1] 1.570796
atan(0.5)
## [1] 0.4636476

R pre-defines several useful constants. pi is one. Others include letters, LETTERS, month.name, month.abb. Try them below:

letters
month.name

Scientific Notation

tan(pi / 2) above returned 1.633124e+16. This is scientific notation: a real coefficient \(a\) (\(1 \leq |a| < 10\)) times an integer power of 10. The letter e connects the coefficient and the exponent. So \(200 = 2 \times 10^2\) is written 2e2, and \(0.002 = 2 \times 10^{-3}\) is 2e-3.

Boolean Algebra

R uses TRUE / FALSE (or T / F) for the two truth values. In numeric contexts, FALSE becomes 0 and TRUE becomes 1. The three Boolean connectives are & (and), | (or), ! (not). The Boolean empty set is NULL.

Boolean expressions pair with comparison operators: ==, !=, >, <, >=, <=.

# Equalities
1 == 2
## [1] FALSE
1 == TRUE   # TRUE and FALSE behave as 1 and 0 with logical attributes
## [1] TRUE
0 != F
## [1] FALSE
# Inequalities
1 != 2
## [1] TRUE
1 > 2
## [1] FALSE
1 < 2
## [1] TRUE
1 >= 2
## [1] FALSE
1 <= 2
## [1] TRUE
# Boolean operations
3 > 2 & 2 > 1
## [1] TRUE
12 & 2 > 2
## [1] FALSE
-3 | 2 > 2
## [1] TRUE

In 3 > 2 & 2 > 1, R first evaluates the comparisons, then the connective. When a single value enters a Boolean expression — as in 12 & 2 > 2 — R treats it as a self-equality check.

Why the double equal sign?

In R, the single = is reserved for assignment, so equality requires ==. This keeps it consistent with !=, >=, <=. Forgetting and writing = is normal at first — practice will fix it.

Vectors

In R, c() (“concatenate”) builds vectors. For consecutive integers, the colon : is a shortcut. The sequence function seq() is the third option.

# Method 1: c()
a  <- c(11, 12, 13, 14, 15)

# Method 2: colon (consecutive integers only)
a1 <- 11:15

# Method 3: seq()
a2 <- seq(from = 11, to = 15, by = 1)
a3 <- seq(11, 15)

# All three match the original
a == a1
## [1] TRUE TRUE TRUE TRUE TRUE
a == a2
## [1] TRUE TRUE TRUE TRUE TRUE
a == a3
## [1] TRUE TRUE TRUE TRUE TRUE

Indexing

Square brackets [] extract elements from a vector.

a <- c(11, 12, 13, 14, 15)

# Single element
a[2]
## [1] 12
a[5]
## [1] 15
# Multiple elements
a[c(3, 4)]
## [1] 13 14

The assignment arrow <-

<- is R’s standard assignment operator. The arrow points left: the right-hand value is stored into the left-hand name.

R is essentially the only popular language using <-; others use =. The reason traces back to APL (A Programming Language) in the 1960s, whose dedicated keyboard had a left-arrow key in the upper-right corner. APL faded, but the <- habit persisted into S, and then into R.

Keyboard shortcut: Alt + - on Windows / Linux, Option + - on macOS.

Vector Arithmetic

Addition and subtraction combine corresponding elements:

\[\vec{a} \pm \vec{b} = (a_1 \pm b_1,\; a_2 \pm b_2,\; \ldots,\; a_n \pm b_n).\]

a <- c(11, 12, 13, 14, 15)

# Vector ± scalar
b <- a + 4
c <- a - 0.5

# Vector ± vector
a + b
## [1] 26 28 30 32 34
a - c
## [1] 0.5 0.5 0.5 0.5 0.5

For multiplication, * is the element-wise (Hadamard) product, %*% is the inner (dot) product, and %o% (or outer()) is the outer (tensor) product.

a <- c(11, 12, 13, 14, 15)
b <- a + 4

# Element-wise (Hadamard) product
a * b
## [1] 165 192 221 252 285
# Inner (dot) product
a %*% b
##      [,1]
## [1,] 1115
# Outer (tensor) product
a %o% b
##      [,1] [,2] [,3] [,4] [,5]
## [1,]  165  176  187  198  209
## [2,]  180  192  204  216  228
## [3,]  195  208  221  234  247
## [4,]  210  224  238  252  266
## [5,]  225  240  255  270  285

Vector “non-standard” operations

Mathematically, vectors have no division. In R, a / b performs element-wise division.

a <- 11:15
b <- a + 4

a / b
## [1] 0.7333333 0.7500000 0.7647059 0.7777778 0.7894737

When two vectors have unequal length, R applies the recycling rule: the shorter vector is reused in cycles. With e <- c(1, 2) and a of length 5, e becomes (1, 2, 1, 2, 1).

a <- 11:15
e <- c(1, 2)

a * e
## [1] 11 24 13 28 15

R also issues a warning.

c() can build character vectors — wrap each element in double quotes:

aSentence <- c("I", "am", "from", "People's Republic of China", ".")
aSentence
## [1] "I"                          "am"                        
## [3] "from"                       "People's Republic of China"
## [5] "."

Exercise 2. Derive the sixty pairs of the Sexagenary Cycle.

China’s traditional calendar pairs ten Heavenly Stems with twelve Earthly Branches. Of the 120 possible combinations, only sixty are canonical: Yang stems (odd positions) pair only with Yang branches; Yin stems (even positions) only with Yin branches.

  • Heavenly Stems: 甲, 乙, 丙, 丁, 戊, 己, 庚, 辛, 壬, 癸
  • Earthly Branches: 子, 丑, 寅, 卯, 辰, 巳, 午, 未, 申, 酉, 戌, 亥

Hints

  • seq() with %% separates Yang from Yin.
  • outer() generates all pairwise combinations.
  • paste0() concatenates two strings (e.g. paste0("甲", "子") returns "甲子").
tiangan <- c("甲", "乙", "丙", "丁", "戊", "己", "庚", "辛", "壬", "癸")
dizhi   <- c("子", "丑", "寅", "卯", "辰", "巳", "午", "未", "申", "酉", "戌", "亥")
tiangan <- c("甲", "乙", "丙", "丁", "戊", "己", "庚", "辛", "壬", "癸")
dizhi   <- c("子", "丑", "寅", "卯", "辰", "巳", "午", "未", "申", "酉", "戌", "亥")

tg_yang <- tiangan[seq(tiangan) %% 2 == 1]
tg_yin  <- tiangan[seq(tiangan) %% 2 == 0]

dz_yang <- dizhi[seq(dizhi) %% 2 == 1]
dz_yin  <- dizhi[seq(dizhi) %% 2 == 0]

outer(tg_yang, dz_yang, FUN = paste0)
outer(tg_yin,  dz_yin,  FUN = paste0)

Matrices

Vectors are one-dimensional; matrices extend to two dimensions.

\[ A_{2 \times 2} = \begin{pmatrix} 1 & 3 \\ 2 & 4 \end{pmatrix}. \]

# 2-row matrix; data fill column by column
matrix(c(1, 2, 3, 4), nrow = 2)
##      [,1] [,2]
## [1,]    1    3
## [2,]    2    4

nrow is “number of rows”; ncol is “number of columns”. Specify only one — R computes the other. R fills matrices in column-major order. The recycling rule also applies during construction:

matrix(1:3, nrow = 2)
##      [,1] [,2]
## [1,]    1    3
## [2,]    2    1

Elements are referenced by row and column. A blank position means “all”.

A <- matrix(1:3, nrow = 2)

A[1, 2]
## [1] 3
A[1, ]
## [1] 1 3
A[, 2]
## [1] 3 1

Single-Matrix Operations

Transposition swaps rows and columns; in R, t().

A <- matrix(1:3, nrow = 2)

A
##      [,1] [,2]
## [1,]    1    3
## [2,]    2    1
t(A)
##      [,1] [,2]
## [1,]    1    2
## [2,]    3    1
t(t(A))
##      [,1] [,2]
## [1,]    1    3
## [2,]    2    1

Nested function calls

t(t(A)) evaluates from the inside out: R first runs t(A), then applies the outer t() to that result. This is identical in spirit to mathematical function composition \(g(f(x))\).

Matrix Operations

Matrix operations parallel their vector analogues. Addition and subtraction require matching dimensions. Multiplication has element-wise, inner, outer, and cross-product forms (crossprod() computes \(X^{\!\top} X\) or \(X^{\!\top} Y\)).

B <- matrix(1:6, nrow = 2)
C <- matrix(6:1, nrow = 2)
D <- matrix(6:1, nrow = 3)

B
##      [,1] [,2] [,3]
## [1,]    1    3    5
## [2,]    2    4    6
C
##      [,1] [,2] [,3]
## [1,]    6    4    2
## [2,]    5    3    1
D
##      [,1] [,2]
## [1,]    6    3
## [2,]    5    2
## [3,]    4    1
# Matching dimensions required
B + C
##      [,1] [,2] [,3]
## [1,]    7    7    7
## [2,]    7    7    7
B - C
##      [,1] [,2] [,3]
## [1,]   -5   -1    3
## [2,]   -3    1    5
# Multiplication varieties
B * C            # element-wise (Hadamard)
##      [,1] [,2] [,3]
## [1,]    6   12   10
## [2,]   10   12    6
B %*% D          # inner (dot) product

Solving Linear Systems

Many statistical methods reduce to solving linear systems. Consider:

\[ \begin{cases} a + b + c = 6, \\ 3a + 2b + 4c = 9, \\ 2a + 2b - 6c = 4. \end{cases} \]

Letting B hold the coefficients, \(\vec{y}\) the right-hand side, \(\vec{x}\) the unknowns: \(\vec{y} = \mathbf{B}\vec{x}\). This resembles a regression equation \(y = \beta_0 + \beta_1 x_1 + \cdots + \beta_n x_n + \epsilon\). In R, solve() returns the answer:

B <- matrix(c(1, 3, 2, 1, 2, 2, 1, 4, -6), nrow = 3)
B
##      [,1] [,2] [,3]
## [1,]    1    1    1
## [2,]    3    2    4
## [3,]    2    2   -6
y <- c(6, 9, 4)

x <- solve(B, y)
x
## [1] -5 10  1
